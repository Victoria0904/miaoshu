# 喵舒 · 女性生命力养护湾

> 面向进行性加重痛经女性的 AI 疼痛评估与就医导航工具

---

## 项目简介

**喵舒**是面向进行性加重痛经、非经期盆腔痛女性的 AI 疼痛评估与就医导航工具，帮助女性在"识别 → 记录 → 评估 → 缓解 → 复盘 → 就医导航"的完整养护链路上，重新掌控自己的身体节奏与好看时间。

### 解决什么问题

子宫内膜异位症、腺肌症引起的疼痛正在系统性偷走女性本该投资于自我养护的「好看时间」。现有工具只记录日期或泛化数据，不解决"内异/腺肌痛经女性如何陪伴式应对疼痛"的问题。

### 核心功能（6 个）

| # | 功能              | 核心价值                        |
| - | --------------- | --------------------------- |
| 1 | 疼痛记录（含人体疼痛点位标记） | 结构化记录周期性疼痛 + 非经期隐痛，精准标记疼痛位置 |
| 2 | 疼痛评估            | VAS 量化 + 趋势对比 + 异常信号提醒      |
| 3 | AI 疼痛舒缓         | 发作时 30 秒内生成个性化非药物舒缓方案       |
| 4 | 生命力复盘           | 量化"被偷走的好看时间" + 五种时间分布图      |
| 5 | 就医报告导出          | 结构化报告 + PDF 下载/微信保存         |
| 6 | AI 就医导航         | 识别进行性加重模式，提示就医参考信号，缩短就医决策延迟 |

### 产品边界

- ✅ **做**：疼痛记录、疼痛规律与触发因素识别、个性化疼痛期养护方案、生命力周期复盘、就医沟通辅助、生活节奏建议
- ❌ **不做**：医疗诊断、疾病筛查、具体治疗建议、用药推荐、医院/科室推荐、中医辨证论治

## 技术栈

| 层级         | 技术选型                               | 说明                                           |
| ---------- | ---------------------------------- | -------------------------------------------- |
| **前端**     | 微信小程序原生（JS ES2020+）                | 后续可迁移 Taro/uni-app 跨平台                       |
| **UI 组件**  | WeUI Mini Programs 2.x             | 微信官方组件库                                      |
| **图表**     | ec-canvas（ECharts 微信小程序版）          | 环形图、趋势曲线                                     |
| **状态管理**   | mobx-miniprogram                   | 轻量响应式                                        |
| **后端**     | Supabase（云托管 BaaS）                 | PostgreSQL + Auth + Storage + Edge Functions |
| **数据库**    | PostgreSQL v16（Supabase 内置）        | JSONB 支持半结构化数据                               |
| **用户认证**   | 微信登录 → Supabase Auth               | RLS 行级安全隔离用户数据                               |
| **AI 大模型** | GLM-5.2（智谱，首推）/ Kimi K2.7 Code（备选） | SSE 流式输出                                     |
| **PDF 生成** | 服务端 Puppeteer                      | 通过 Supabase Edge Function 调用                 |
| **部署**     | Supabase 云托管（AWS 新加坡节点）            | 国内延迟 \~150ms                                 |

## 如何运行

### 环境准备

1. **微信开发者工具**（最新稳定版）
2. **Node.js** 16+（如需构建脚本）
3. **Supabase 账号**（免费版即可）

### 本地开发

```bash
# 1. 克隆项目
git clone <repo-url>
cd miaoshu

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入以下配置：
# - SUPABASE_URL=your_supabase_project_url
# - SUPABASE_ANON_KEY=your_supabase_anon_key
# - AI_API_KEY=your_glm_or_kimi_api_key
# - AI_API_ENDPOINT=https://open.bigmodel.cn/api/paas/v4

# 3. 用微信开发者工具打开项目根目录
# 4. 在工具中填入你的 AppID（或使用测试号）
# 5. 点击「编译」即可预览
```

### Mock 模式

开发阶段无需连接真实后端，所有 service 函数支持 `USE_MOCK` 开关：

```javascript
// services/mock-data.js
const USE_MOCK = true;  // 改为 false 切换到真实 API
```

### 数据库初始化

在 Supabase SQL Editor 中执行建表语句（详见 [ARCHITECTURE.md](ARCHITECTURE.md) 第五节）。
