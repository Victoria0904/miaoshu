# 喵舒 · 女性生命力养护湾 · 整体开发技术方案

> **版本：** v1.0  
> **日期：** 2026-07-20  
> **适用场景：** 微信小程序 MVP 正式产品雏形  
> **核心原则：** 刚刚好 —— 满足 6 个 P0 功能闭环，预留扩展接口，不为远期功能增加当前复杂度。

---

## 1. 项目目标与范围

### 1.1 项目定位

喵舒 · 女性生命力养护湾：面向进行性加重痛经、非经期盆腔痛女性的 **AI 疼痛评估与就医导航工具**，帮助用户在「识别 → 记录 → 评估 → 缓解 → 复盘 → 就医导航」的完整养护链路上，重新掌控自己的身体节奏与好看时间。

### 1.2 MVP 功能边界（P0）

| 编号 | 功能 | 核心交付物 | 边界说明 |
| :--: | --- | --- | --- |
| F1 | 疼痛记录（含人体疼痛点位标记） | 人体图标记 + 表单 + 日历/列表双视图 | 仅记录，不做医学判断 |
| F2 | 疼痛评估 | VAS 0-10 滑块 + 当次分级 + 历史均值对比 | 仅当次评估 |
| F3 | AI 疼痛舒缓 | 场景选择 → 流式/异步生成舒缓方案 | 非药物、非医疗方案 |
| F4 | 生命力复盘 | 五种时间环形图 + 趋势曲线 + AI 减损建议 | 基于模拟/真实记录 |
| F5 | 就医报告导出 | 结构化报告预览 + PDF 占位（Demo 阶段模拟） | 标注「辅助记录，非诊断」 |
| F6 | AI 就医导航 | 跨周期异常信号提示（3 个月趋势识别） | 仅提示「建议就医咨询」 |

### 1.3 不做（业务边界）

- 医疗诊断、疾病筛查、具体治疗建议
- 用药推荐、医院/科室推荐、具体检查建议
- 中医辨证论治
- 未成年人数据收集

## 2. 技术栈选型与理由

| 层级 | 选型 | 版本/说明 | 理由 |
| --- | --- | --- | --- |
| **前端框架** | 微信小程序原生 | 基础库 3.x+ | 目标平台即微信，原生开发最快；后续可迁移 Taro/uni-app |
| **前端语言** | JavaScript (ES2020+) | — | 原生支持，无需编译；关键类型用 JSDoc 描述 |
| **状态管理** | mobx-miniprogram | 最新稳定版 | 轻量响应式，避免 EventBus 混乱 |
| **UI 组件库** | WeUI Mini Programs | 2.x | 微信官方组件，符合平台设计规范 |
| **图表库** | ec-canvas | 2.x | ECharts 小程序版，支持环形图、折线图 |
| **人体图示** | Canvas 2D + SVG 叠加层 | — | 不依赖第三方，支持点击标记与区域高亮 |
| **后端 BaaS** | Supabase | 最新云托管版 | PostgreSQL + Auth + Storage + Edge Functions 零运维 |
| **数据库** | PostgreSQL (Supabase 内置) | v16 | JSONB 支持疼痛点位/症状等半结构化数据 |
| **用户认证** | 微信登录 → Supabase Auth | — | 小程序 wx.login 换 openid → JWT；RLS 隔离 |
| **AI 大模型** | GLM-5.2（主）/ Kimi K2.7 Code（备） | SSE 流式 | 国内合规、长上下文、兼容 Claude Code |
| **PDF 生成** | Puppeteer via Supabase Edge Function | — | 客户端无法生成 PDF，服务端兜底 |
| **文件存储** | Supabase Storage | — | PDF 报告、头像存储 |
| **部署托管** | Supabase Cloud | AWS 新加坡节点 | 国内延迟 ~150ms，后续可迁阿里云/腾讯云 |
| **环境管理** | `.env` + `config.js` | — | API Key、Supabase URL、AI endpoint 统一管控 |

### 2.1 核心架构决策

| 决策 | 选择 | 理由 |
| --- | --- | --- |
| 后端 | BaaS 不自建 | Supabase 覆盖数据、认证、存储、边缘函数，24h 可上线 |
| AI 调用 | 前端直调 SSE（开发期）/ Edge Function 中转（生产期） | MVP 先直调快速验证，生产期加脱敏中转 |
| 数据库 | PostgreSQL + JSONB | 结构化关系 + 半结构化疼痛数据灵活存储 |
| 离线 | 本地缓存优先 + 后台同步 | 疼痛场景下网络可能不稳定，需离线可记录 |

## 4. 后端 / BaaS 架构

### 4.1 Supabase 能力使用

| 能力 | 用途 | MVP 边界 |
| --- | --- | --- |
| Auth | 微信 openid 登录、JWT 会话 | 首次进入强制登录 + 隐私协议 |
| PostgreSQL | 用户资料、疼痛记录、舒缓方案、复盘、报告 | RLS 隔离 |
| Storage | PDF 报告、用户头像 | 报告占位阶段不上传 |
| Edge Functions | AI 中转脱敏、PDF 生成、复杂数据聚合 | 第二阶段启用 |

### 4.2 数据模型

#### 4.2.1 用户表 `profiles`

```sql
CREATE TABLE profiles (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  openid      TEXT UNIQUE NOT NULL,
  nickname    TEXT,
  avatar_url  TEXT,
  age_range   TEXT,              -- '18-25' | '26-35' | '36-45' | '46+'
  constitution_tag TEXT,          -- 中医体质标签
  cycle_length INT,              -- 经期周期天数
  period_length INT,             -- 经期持续天数
  last_period_date DATE,
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);
```

#### 4.2.2 疼痛记录表 `pain_records`

```sql
CREATE TABLE pain_records (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES profiles(id),
  pain_intensity  INT NOT NULL,      -- 0-10
  pain_type       TEXT,              -- '钝痛' | '刺痛' | '绞痛' | '坠痛' | '隐痛' | '灼烧痛'
  pain_locations  JSONB,             -- [{x, y, intensity, zone, label}]
  duration_start  TIMESTAMPTZ,
  duration_end    TIMESTAMPTZ,
  symptoms        JSONB,             -- ['恶心', '呕吐', ...]
  triggers        JSONB,             -- ['饮食', '运动', ...]
  relief_methods  JSONB,             -- ['热敷', '休息', '用药'] + 效果
  medications     JSONB,             -- [{name, dosage, time, effect}]
  period_day      INT,               -- 经期第几天，null = 非经期
  cycle_phase     TEXT,              -- menstrual | follicular | ovulation | luteal
  scene           TEXT,              -- home | office | commute | travel
  ai_analysis     JSONB,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

#### 4.2.3 AI 舒缓方案表 `relief_plans`

```sql
CREATE TABLE relief_plans (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES profiles(id),
  pain_record_id UUID REFERENCES pain_records(id),
  scene           TEXT,
  plan_content    JSONB,             -- [{action, duration, difficulty, area}]
  avoid_items     JSONB,
  time_summary    TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

#### 4.2.4 月度复盘表 `monthly_reviews`

```sql
CREATE TABLE monthly_reviews (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES profiles(id),
  month       DATE NOT NULL,
  pain_hours  DECIMAL(5,2),
  time_breakdown JSONB,              -- {survival, earning, beauty, fun, flow}
  stolen_beauty_hours DECIMAL(5,2),
  ai_suggestion TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, month)
);
```

#### 4.2.5 就医报告表 `medical_reports`

```sql
CREATE TABLE medical_reports (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES profiles(id),
  date_range_start DATE,
  date_range_end   DATE,
  report_content JSONB,
  pdf_url        TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

### 4.3 RLS 策略示例

```sql
ALTER TABLE pain_records ENABLE ROW LEVEL SECURITY;

CREATE POLICY "用户只能查看自己的疼痛记录" ON pain_records
  FOR SELECT USING (user_id = auth.uid());

CREATE POLICY "用户只能创建自己的疼痛记录" ON pain_records
  FOR INSERT WITH CHECK (user_id = auth.uid());
```

## 5. AI 能力设计

### 5.1 AI 应用场景

| 场景 | 输入 | 输出 | 模型 |
| --- | --- | --- | --- |
| AI 疼痛舒缓 | 症状、疼痛点位、强度、场景、体质标签 | 分钟级非药物舒缓方案 | GLM-5.2 SSE |
| 就医报告摘要 | 近 3-6 个月疼痛记录 | 主诉、现病史、异常信号、就医问题清单 | GLM-5.2 |
| 生命力复盘建议 | 月度疼痛数据、时间分布 | 减损建议、恢复计划 | GLM-5.2 |
| 就医导航信号 | 跨周期疼痛强度、频次、点位 | 是否提示「建议就医咨询」 | 规则 + 轻量统计 |

### 5.2 Prompt 边界

- **System Prompt**：明确角色为「非医疗生活方式陪伴助手」，禁止诊断、用药、检查建议。
- **User Prompt**：结构化 JSON，仅含疼痛强度、部位、场景、体质标签等，脱敏姓名/手机号/位置。
- **输出格式**：强制 JSON Schema，包含 `actions`、`avoid_items`、`time_summary`。

### 5.3 流式输出

- 前端使用 `wx.request` 开启 `enableChunked: true` 接收 SSE。
- 或使用 `fetch` + `ReadableStream`（小程序基础库 2.14.4+）。
- 流式文本逐字渲染，提升等待体验。

## 6. 核心功能实现方案

### 6.1 疼痛记录（F1）

- **人体图**：Canvas 绘制正面/背面轮廓，SVG 叠加 clickable zone。
- **标记存储**：归一化坐标 (0-1)，适配不同屏幕。
- **表单**：VAS 滑块、疼痛性质单选、伴随症状多选、场景单选。
- **日历视图**：基于 `new Date()` 生成网格，叠加经期标记、疼痛色块、非经期疼痛点。
- **保存**：先写本地缓存，后台异步同步 Supabase；MVP 阶段可完全本地存储 + Mock。

### 6.2 疼痛评估（F2）

- 单次 VAS 0-10 滑块。
- 与历史均值对比（本地计算）。
- 强度 ≥ 7 时，UI 提示「疼痛剧烈，建议即时关注」。

### 6.3 AI 疼痛舒缓（F3）

- 入口：首页「正在疼痛？」+ 记录页保存后推荐。
- 流程：选择场景 → 构造 prompt → SSE 调用 AI → 解析 JSON → 展示动作卡片 + 避免事项。
- MVP Mock：提供静态 JSON 方案，覆盖 4 种场景。

### 6.4 生命力复盘（F4）

- 数据聚合：当月疼痛记录 → 计算疼痛总时长、平均强度、发作次数。
- 五种时间：基于「被偷走的好看时间」反推，其他时间用默认值或用户设置。
- 图表：ec-canvas 环形图 + 折线图。
- AI 建议：基于聚合数据 + prompt 生成。

### 6.5 就医报告导出（F5）

- 选择时间范围（近 1/3/6 个月）。
- 聚合：主诉、现病史、疼痛曲线、用药记录、缓解方式。
- AI 生成摘要 + 问题清单。
- MVP PDF：先展示「生成中」+ 模拟下载，第二阶段接入 Puppeteer。

### 6.6 AI 就医导航（F6）

- 规则引擎：近 3 个月平均强度上升、非经期疼痛频次增加、强度突破个人基线。
- 触发后在首页/我的页显示「AI 就医参考信号」卡片。
- 文案严格：「建议关注并就医咨询」，不涉及疾病名称。

## 10. 安全、合规与免责声明

### 10.1 数据安全

- Supabase RLS 强制隔离用户数据。
- TLS 1.3 传输加密，静态 AES-256。
- AI 调用前脱敏：不传用户名、手机号、精确位置。

### 10.2 合规边界

- 隐私协议首次启动强制展示。
- 每个 AI 建议页面常驻「AI 建议非医疗诊断」免责声明。
- 就医报告顶部标注「辅助记录，非诊断」。
- 剧烈疼痛（≥8 分）或异常症状时弹出紧急就医提醒。

### 10.3 文案红线

| 不当表述 | 推荐表述 |
| --- | --- |
| 判断是否需要就医 | 识别就医参考信号 / 建议关注并就医咨询 |
| 该查 B 超 | 建议根据医生指导进行相关检查 |
| 挂什么科 | 仅提供常见科室信息供参考 |
| 原发/继发鉴别 | 疼痛模式识别与异常信号提醒 |
